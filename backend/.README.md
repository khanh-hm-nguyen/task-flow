## üíæ Database Implementation
The project supports two database modes. You can toggle these in `src/main/resources/application.properties`.

### Option A: PostgreSQL (Recommended for Docker)
The project includes a `docker-compose.yml` to spin up a PostgreSQL instance.
1.  Ensure you have a `.env` file with `DB_NAME`, `DB_USERNAME`, and `DB_PASSWORD`.
2.  Run: `docker-compose up -d`
3.  Uncomment the PostgreSQL lines in `application.properties`.

### Option B: H2 In-Memory (Currently Active)
For quick testing without setup, the project is currently configured to use **H2**.
* **Console**: Access at `http://localhost:8080/h2-console`
* **JDBC URL**: `jdbc:h2:mem:testdb`
* **User**: `sa` | **Password**: (empty)

## üîå API Reference
All endpoints below require a valid JWT Token passed in the Authorization: Bearer <token> header. The backend automatically filters results so you only interact with data you own.
The backend exposes the following REST endpoints:

```text
Method      Endpoint                                   Description
GET         /task-lists                                Get all task lists
POST        /task-lists                                Create a new task list
DELETE      /task-lists/{id}                           Delete a task list
GET         /task-lists/{id}/tasks                     Get all tasks for a list
POST        /task-lists/{id}/tasks                     Create a new task
PUT         /task-lists/{listId}/tasks/{taskId}        Update a task (Status, Title, etc.)
DELETE      /task-lists/{listId}/tasks/{taskId}        Delete a task
```

### üîë Authentication Endpoints
These routes are public and do not require a token to access.

```text
Method      Endpoint                                   Description
POST        /api/auth/register                         Register a new user and return an initial JWT.
POST        /api/auth/login                            Validate credentials and return a fresh JWT.
```
